# Contributing to YAPL

Thanks for your interest in improving YAPL!

## Building

Requirements: JDK 21 and the Android SDK (compile/target SDK 35).

```bash
./gradlew assembleDebug        # debug APK (be.moraine.yapl.debug)
./gradlew assembleRelease      # minified release APK (be.moraine.yapl)
./gradlew testDebugUnitTest    # unit tests
```

## Project layout

Clean Architecture in three layers under `app/src/main/java/dev/pilotlog/`:

- `domain/` — models, repository interfaces and use cases (pure Kotlin, unit-tested).
- `data/` — Room database, DAOs, entities, mappers, repository implementations.
- `ui/` — Jetpack Compose (Material 3) screens, view models and navigation.

The bundled airport database is generated by `scripts/build_airports_db.py` from
`res/airports.dat` (OurAirports). Room schemas are exported to `app/schemas/`.

## Ground rules

- **Never commit personal data.** Real flight logs, backups, exported CSV/PDF and
  logbook scans must stay out of the repository — they are gitignored (`legacy/`,
  `*-flights.csv`, `yapl-logbook*.pdf`). Double-check `git status` before committing.
- Keep the app **permission-free and offline** — no network, analytics or tracking
  dependencies (this is required for F-Droid inclusion).
- Add a database migration and bump the Room version for any schema change, and
  keep the exported schema JSON in `app/schemas/`.
- New source files carry the SPDX header `// SPDX-License-Identifier: GPL-3.0-only`.
